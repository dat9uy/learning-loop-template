# Phase 1: R2 Write-Gate (per-runtime allowlist) + identity pinning

## Overview

Add a per-runtime write allowlist gate (`checkR2Ownership`) inside `createLoopTool`'s factory seam, plus a process-boot identity pin (`getPinnedRuntimeId`) that replaces LIM-3's role without the crypto. Every MCP tool flows through this gate before `execute` runs.

This phase also wires **the 4 must-fix findings** from the red-team review (R1-R4) — without them, the gate ships with CRITICAL/HIGH exploitable gaps.

## Threat model (what this phase defends against)

| Threat | Concrete attack | This phase defends? |
|---|---|---|
| Cross-runtime surface write | Runtime A's process writes to Runtime B's surface dir | **Yes** (R2 ownership check) |
| Allowlist self-bootstrap | Tool rewrites `.loop/r2-allowlist.json` to grant itself everything | **Yes** (R1 hard-deny + `update_r2_allowlist` tool) |
| Path-bearing arg smuggling | Tool smuggles a write path through `metadata.file` or array arg | **Yes** (R3 path-field detector with default-deny) |
| Workflow tool bypass | Tool runs inside `convertWorkflowsToTools` which uses raw `createTool` | **Yes** (R4 swap to `createLoopTool`) |
| Closure-variable mutation | Future refactor swaps closure for settable property | **Yes** (R2 `Object.freeze` + immutability test) |

This phase does NOT defend against (documented residual; see Phase 3 §Out-of-scope):
- Read-then-write composition via tool stdout (no defense in plan scope)
- Subprocess-spawn re-pin detection (deferred to a future plan)
- Surface-registry spoofing via boot-time env injection (deferred to LIM-3 follow-up plan)

- **F4.** Per-surface mapping (stub F4):

- **F6.** Gate emits a `gate_log` row on every denial (allowlist miss OR override use). The log entry's JSON-serialized form MUST NOT contain raw `\n` or `\r` (Phase 3 R6 hardening). (Stub F6 + R6 partial)
- **F7.** `tools/learning-loop-mastra/core/surfaces.js:16` `SURFACES` constant is extended to `[".claude", ".factory", ".mastracode"]`. Single source of truth for the audit-log gate + R2 gate; no divergence. (Stub F7)

#### Red-team must-fix additions (F8-F14)

- **F8. (R1)** **Self-bootstrap hard-deny:** `checkR2Ownership` MUST deny writes to `.loop/r2-allowlist.json` for ALL runtimes (including the one that "owns" the file) BEFORE the allowlist lookup. The deny message names `update_r2_allowlist` MCP tool as the legitimate path. The same hard-deny list applies to `runtime-state.jsonl` and `.gate-override` (per F9 + R6 + R17). Implementation: a static `BOOTSTRAP_DENY_PATTERNS` constant at the top of `core/r2/ownership.js`, evaluated first. Test: `__tests__/r2/ownership.test.js#allowlist_self_write_denied`.
- **F9. (R1)** **`update_r2_allowlist` MCP tool:** operator-only path to edit `.loop/r2-allowlist.json`. Requires `gate_mark_preflight({surface: "operator"})` (existing pattern in `core/evaluate-write-gate.js`). Writes via temp-file + rename (atomic; matches `surfaces.js:39-58` pattern). Validates against schema before rename. Triggers allowlist cache invalidation (re-read on next R2 call). Logs `gate_log` entry BEFORE the rename (Phase 3 R6 ordering).
- **F10. (R2)** **Pinned runtime ID is immutable:** `pinnedRuntimeId` is wrapped in `Object.freeze({ readonly runtime })`. `getPinnedRuntimeId()` returns the frozen object's `runtime` field. NO `setPinnedRuntimeId` export exists in `mastra/server.js` or `create-loop-tool.js`. The closure-variable shape is asserted by `__tests__/pin-runtime-id.test.js#closure_immutability` (mid-process `process.env.LOOP_SURFACE = '.factory'` does NOT flip the pin; direct property assignment throws in strict mode).
- **F11. (R3)** **Path-field detector with default-deny:** new file `core/r2/path-field-detector.js` exports `detectPathFields({ tool, args })` that returns `Set<string>` of path-bearing values from `args`. Behavior:
  - Reads the tool's declared `pathFields` from `tools/manifest.json` (new field; every entry MUST have `pathFields: string[]` — see F12).
  - Recursively scans declared `pathFields` paths up to depth 3.
  - Array args: each element is scanned.
  - Default-deny: if `pathFields` is undefined at boot, the server throws `path_fields_undefined_for_tool` and exits (loud failure).
  - Explicit opt-out: `pathFields: []` declares "no path args" (returns empty set; gate short-circuits to allow).
  - Test: `__tests__/r2/path-field-detector.test.js` covers recursion, array, default-deny, opt-out.
- **F12. (R3)** **`pathFields` declared at tool registration:** every entry in `tools/manifest.json` MUST have a `pathFields: string[]` field. Empty array `[]` is allowed (declares "no path args"). Missing field → boot throws. The contract validator (Phase 3 C2) adds Req #11: `tools-manifest-has-path-fields`.
- **F13. (R4)** **Workflow + agent tools flow through R2:** modify `mastra/server.js#convertWorkflowsToTools` (lines 78-161) to swap `createTool` for `createLoopTool`. This routes workflow tools through R2 + path-containment. Agent tools are already routed through `createLoopTool` per `agents-manifest.json` (verified). Test: `__tests__/r2/workflow-coverage.test.js` registers a workflow with a write step targeting `.factory/**` from a Claude session → throws `cross_runtime_write_denied`.
- **F14. (R9 + R10)** **Path normalization + locked glob semantics:** before `globMatch`, call `path.resolve(root, userPath)` + `path.normalize()` (resolves `./`, `..`, leading slashes). Then `realpath(userPath)` (symlink resolution; LIM-4 covers hardlinks). The glob semantics (`**` matches `.*`, `*` matches `[^/]*`) are locked by `__tests__/r2/glob-match.test.js` cases. Patterns like `.claude**` (no slash before `**`) emit a startup warning (`warn: ambiguous_glob_pattern`).

### Non-functional

- **NF1.** Allowlist loaded once at server boot (cached for process lifetime). Operator edits via `update_r2_allowlist` tool trigger cache invalidation + re-read on next call. **No automatic file-watching** to keep the contract simple; operators must explicitly call the tool. (Stub NF1)
- **NF2.** Gate logic adds ≤ 0.5ms per tool call: single `JSON.parse` at boot; in-memory glob match per call via `RegExp` translation (existing `core/gate-logic.js#globMatch` at lines 47-58). Path normalization adds ≤ 0.1ms.
- **NF3.** Denials are fail-closed: any gate exception → deny + log + escalate (GATE_RESPONSE_MODE=escalate path). (Stub NF3)
- **NF4. (R2)** Boot-time `pinRuntimeIdAtBoot()` is **synchronous** and called as the FIRST executable statement in `mastra/server.js` (before any `await` or `import` resolution that could fire `beforeExit` hooks). Verified by `__tests__/pin-runtime-id.test.js#sync_pin` case.

## Architecture

### Gating chain (every tool call)

```
Tool call arrives
  → withR2Gate({ id, args, execute })
      1. resolveSafePath(root, args[pathField]) ← LIM-4 (Phase 2)
      2. detectPathFields(args)              ← R3 (this phase)
      3. checkR2Ownership({ runtime, paths }) ← core R2 (this phase)
           - static BOOTSTRAP_DENY first (R1: .loop/, runtime-state.jsonl, .gate-override)
           - path normalize + glob match
           - return { allowed, reason }
      4. if denied → appendGateLog + throw cross_runtime_write_denied
  → execute(args)
```

### Identity pinning (replaces LIM-3's role in `createLoopTool`)

```js
// tools/learning-loop-mastra/mastra/server.js (top of file, BEFORE any await)
import { SURFACES } from "../core/surfaces.js";
import { readFileSync } from "node:fs";
import { join } from "node:path";

const SURFACE_TO_RUNTIME = {
  ".claude": "claude-code",
  ".factory": "droid",
  ".mastracode": "mastra-code",
};

const pinErrorConstants = readFileSync(
  join(import.meta.dirname, "identity-errors.json"),
  "utf8",
);

// Frozen, immutable, closure-scoped. NO export. NO setter.
const pinState = Object.freeze({
  runtime: (() => {
    const surface = process.env.LOOP_SURFACE;
    if (!surface) {
      throw new Error(pinErrorConstants.MISSING_LOOP_SURFACE);
    }
    if (!SURFACES.includes(surface)) {
      throw new Error(
        pinErrorConstants.INVALID_LOOP_SURFACE
          .replace("{value}", surface)
          .replace("{allowed}", SURFACES.join(", "))
      );
    }
    const runtime = SURFACE_TO_RUNTIME[surface];
    if (!runtime) {
      throw new Error(pinErrorConstants.MISSING_RUNTIME_MAPPING.replace("{surface}", surface));
    }
    return runtime;
  })(),
});

export function getPinnedRuntimeId() {
  return pinState.runtime;  // throws if un-pinned (process-boot failure)
}
```
