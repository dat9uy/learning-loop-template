## Architecture

**Substrate decisions (locked from research report §1, §2, §3):**

- **Package layout:** flat single-package — no `packages:` key in `pnpm-workspace.yaml`. The legacy server is not a workspace member; the new one matches.
- **Import aliases:** add `#mastra/*` to root `package.json#imports` (alongside the existing `#mcp/*` + `#lib/*`).
- **Stub tool:** one `createTool` returning `{ ok: true }` — enough to verify `MCPServer` boots + answers `tools/list`. Replaced in Phase 2.
- **Stdio transport:** `await server.startStdio()` — one line, no manual `StdioServerTransport` wiring. Verified at `mastra.ai/guides/guide/publishing-mcp-server`.
- **Invocation:** `node tools/learning-loop-mastra/server.js` (matches legacy pattern at `package.json:18`).

```js
// tools/learning-loop-mastra/server.js (Phase 0 stub)
import { MCPServer } from "@mastra/mcp";
import { createTool } from "@mastra/core/tools";

const stubTool = createTool({
  id: "mastra_stub",
  description: "Stub tool for Phase 0 boot verification. Replaced in Phase 2.",
  inputSchema: { type: "object", properties: {} },
  execute: async () => ({ ok: true }),
});

const server = new MCPServer({
  id: "learning-loop-mastra",
  name: "learning-loop-mastra",
  version: "0.1.0",
  description: "Mastra-based peer MCP server for the learning loop (Phase C Plan 1)",
  tools: { mastra_stub: stubTool },
});

await server.startStdio();
```

## Implementation Steps

**Step 1 — Branch + baseline (~5 min)**
1. `git checkout main && git pull`

1. Edit `package.json` to add `@mastra/core` and `@mastra/mcp` to `dependencies`. Use the most recent stable minor verified by `mastra.ai` (check the project's `~/.cache/mastra` cache or the embedded docs if `@mastra/core` is already in any node_modules tree).
