# Phase 4: Per-worktree .loop-version file (gitignored)

## Overview

Each worktree declares its L2 version in `<root>/.loop-version`. The file is generated lazily on first read and gitignored (same pattern as `.last-operator-message`). Two purposes:
1. **Schema-version-skew detection**: a write referencing a `schema_branch` not in the current worktree's `metaStateEntrySchema` discriminator union is rejected with a structured error.
2. **Parallel operation safety**: when worktree A (L2 v2.1) writes an entry with a field shape unknown to worktree B (L2 v2.0), worktree B sees a structured error instead of silently dropping the field.

## Requirements

- **Functional**: `.loop-version` file is created on first registry write if absent. Content is `loop:<major>.<minor>.<patch>\nschema_branches:<comma-separated>\n`. `meta_state_log_change` rejects writes whose `change_diff` references an unknown schema branch.
- **Non-functional**: First-write overhead is <10ms (file creation + sha256 hash).

## Architecture

`tools/learning-loop-mastra/core/worktree-version.js` (new, ~50 lines):

```js
import { existsSync, readFileSync, writeFileSync } from "node:fs";
import { join } from "node:path";
import { createHash } from "node:crypto";

const VERSION_FILE = ".loop-version";

export function readLoopVersion(root) {
  const path = join(root, VERSION_FILE);
  if (!existsSync(path)) {
    // Lazy creation: derive from package.json + registry sha256
    const initial = deriveInitialVersion(root);
    writeFileSync(path, initial, "utf8");
    return parseVersion(initial);
  }
  return parseVersion(readFileSync(path, "utf8"));
}

function deriveInitialVersion(root) {
  const pkg = JSON.parse(readFileSync(join(root, "package.json"), "utf8"));
  const loopPkg = JSON.parse(readFileSync(join(root, "tools/learning-loop-mastra/package.json"), "utf8"));
  return [
    `loop:${loopPkg.version}`,
    `node:${pkg.engines?.node ?? "unknown"}`,
    `schema_branches:finding,change-log,rule,loop-design`,
    "",
  ].join("\n");
}

function parseVersion(content) {
  const lines = content.split("\n").filter(Boolean);
  const out = {};
  for (const line of lines) {
    const [k, ...rest] = line.split(":");
    out[k.trim()] = rest.join(":").trim();
  }
  if (out.schema_branches) {
    out.schema_branches = out.schema_branches.split(",").map((s) => s.trim());
  }
  return out;
}

export function isSchemaBranchSupported(root, branch) {
  const version = readLoopVersion(root);
  return version.schema_branches?.includes(branch) ?? false;
}
```

`tools/learning-loop-mastra/core/meta-state.js` (writeEntry):

```js
import { isSchemaBranchSupported } from "./worktree-version.js";

export async function writeEntry(root, entry) {
  return enqueue(root, () =>
    withRegistryLock(root, () => {
      // NEW: schema-version-skew detection
      if (entry.entry_kind && !isSchemaBranchSupported(root, entry.entry_kind)) {
        throw new SchemaVersionSkewError(root, entry.entry_kind, readLoopVersion(root));
      }
      // ... existing logic ...
    })
  );
}

export class SchemaVersionSkewError extends Error {
  constructor(root, branch, currentVersion) {
    super(
      `schema_version_skew: entry.entry_kind="${branch}" not in worktree's schema_branches=[${currentVersion.schema_branches?.join(", ")}]. ` +
      `Worktree: ${root}. Update .loop-version if you've migrated to a newer schema.`
    );
    this.code = "SCHEMA_VERSION_SKEW";
    this.branch = branch;
    this.currentVersion = currentVersion;
  }
}
```

## Adversarial context (Finding 7 — supply-chain + Finding 5 — derivation purity)

**Scope limitation:** This phase detects `entry_kind`-discriminator skew only. It does NOT detect per-kind field-shape drift (e.g., `loop-design.proposed_design_for` shape changes between L2 versions). Document this scope explicitly in `docs/architecture.md`; per-kind field validation lands in a follow-up plan.

**Discovery path:** Plan originally derived from `tools/learning-loop-mastra/package.json`. Verified: that file does NOT exist. The actual derivation source is the agent manifest at `tools/learning-loop-mastra/agent-manifest.json` (read tool count + version string). Phase 4 implementation must read from the correct source.

## Risk Assessment

| Risk | Severity | Mitigation |
|---|---|---|
| Existing tests pass `entry_kind: "change-log"` etc. — those are valid branches | Low | `schema_branches` defaults to `finding,change-log,rule,loop-design` (the 4-kind union) |
| Lazy file creation race (2 writers, file doesn't exist) | Low | Phase 1 lock makes it safe; if both writers race, last write wins (file content is identical for same schema_branches) |
| Migration: existing worktrees without `.loop-version` | Low | Lazy creation on first read; no migration needed |
| Test runs leave `.loop-version` in repo | Medium | `.gitignore` covers it; CI test runs use `mkdtempSync` so no pollution |
