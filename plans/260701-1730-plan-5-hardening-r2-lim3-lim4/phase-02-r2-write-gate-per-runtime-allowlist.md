# Phase 2: R2 Write-Gate (per-runtime allowlist)

## Overview

Build the per-runtime write allowlist that keys on the Phase 1 verified `runtime_id`. Each runtime (Claude Code / Droid / Mastra Code) gets an explicit set of writable surfaces (`own` patterns + `universal` patterns); writes outside that set are denied with a structured error and appended to the gate log. **TDD-first**: write the failing allowlist tests BEFORE implementing the gate; the gate wraps `createLoopTool` alongside the Phase 1 identity verifier so every tool that can write a file path is gated by both identity AND ownership.

## Requirements

### Functional

- **F1.** New file `.loop/r2-allowlist.json` (project-local; committed to git) declares each runtime's writable surfaces + universal patterns. Schema version `1`. Generated by `interface/contract.js` from the runtime registry on first server boot; operator can hand-edit.
- **F2.** R2 gate sits AFTER Phase 1's identity verifier in the `createLoopTool` wrapper chain. Order: identity verify → path containment (Phase 3) → R2 ownership → tool execute. Path containment throws first on escape; R2 ownership throws first on cross-runtime write.
- **F3.** Default deny: any path not in `own` AND not in `universal` is denied for that runtime. Override: operator sets `deny: []` per runtime (audit-log entry on every override use).
- **F4.** Per-surface mapping:
  - `claude-code`: own = [`.claude/**`]; deny = [`.factory/**`, `.mastracode/**`].
  - `droid`: own = [`.factory/**`]; deny = [`.claude/**`, `.mastracode/**`].
  - `mastra-code`: own = [`.mastracode/**`]; deny = [`.claude/**`, `.factory/**`].
  - `universal` (all runtimes): [`records/**`, `plans/**`, `docs/**`, `AGENTS.md`, `tools/learning-loop-mastra/**`].
- **F5.** Gate emits structured error on denial: `{ error: "cross_runtime_write_denied", runtime, tool, path, hint, denied_at }`. Hint names the owning runtime and points to `.loop/r2-allowlist.json`.
- **F6.** Gate emits a `gate_log` row on every denial (allowlist miss OR override use).
- **F7.** `tools/learning-loop-mastra/core/surfaces.js:16` `SURFACES` constant is extended to `[".claude", ".factory", ".mastracode"]` (single source of truth for the audit-log gate + R2 gate; no divergence).

### Non-functional

- **NF1.** Allowlist loaded once at server boot (cached for process lifetime). Operator edits require restart. Decision: skip hot-reload (audit-log + boot-time load is simpler; restart is cheap).
- **NF2.** Gate logic adds ≤ 0.5ms per tool call (single `JSON.parse` at boot; in-memory glob match per call via `RegExp` translation; red-team Finding 12: `minimatch` removed).
- **NF3.** Denials are fail-closed: any gate exception → deny + log + escalate (GATE_RESPONSE_MODE=escalate path).

### Gate logic — **REVISED to use built-in `RegExp` (red-team Finding 12)**

```js
// core/r2/ownership.js — REVISED 2026-07-01 per red-team Finding 12 (minimatch NOT in dep tree).
// Use built-in `RegExp` translation with glob-to-regex helper; eliminates dep risk.
// Supports: ** (recursive), * (any not /), ? (single char), literal segments.
import { relative } from "node:path";

function globToRegex(glob) {
  // Convert simple glob to RegExp. Supports `**` (any depth), `*` (any not /), literal text.
  // Sufficient for the allowlist patterns in scope (.claude/**, .factory/**, etc.).
  let re = "";
  for (let i = 0; i < glob.length; i++) {
    const c = glob[i];
    if (c === "*" && glob[i + 1] === "*") {
      re += ".*";
      i++;
    } else if (c === "*") {
      re += "[^/]*";
    } else if (c === "?") {
      re += "[^/]";
    } else if (/[.+^$|(){}\[\]\\]/.test(c)) {
      re += "\\" + c;
    } else {
      re += c;
    }
  }
  return new RegExp("^" + re + "$");
}
```

export function checkR2Ownership(runtimeId, absPath, allowlist, root) {
  const entry = allowlist.runtimes[runtimeId];
  if (!entry) {
    return { ok: false, reason: "unknown_runtime", runtime: runtimeId };
  }
  const rel = relative(root, absPath).replace(/\\/g, "/");
  if (!rel || rel.startsWith("..")) {
    return { ok: false, reason: "outside_root", runtime: runtimeId, path: rel };
  }
  // RED-TEAM FINDING 4: protected paths — NO runtime can write to these, including operator override.
  // (Operator override mechanism is deferred to follow-up; in v1, protected paths are immutable.)
  for (const protectedPat of allowlist.protected_paths ?? []) {
    if (globToRegex(protectedPat).test(rel)) {
      return {
        ok: false,
        reason: "protected_path",
        runtime: runtimeId,
        path: rel,
        hint: `Path '${rel}' is protected. No runtime session may write to it. See .loop/r2-allowlist.json#protected_paths.`,
      };
    }
  }
  // Universal patterns (every runtime may write)
  for (const pat of allowlist.universal ?? []) {
    if (globToRegex(pat).test(rel)) {
      return { ok: true, scope: "universal" };
    }
  }
  // Runtime's own patterns
  for (const pat of entry.own) {
    if (globToRegex(pat).test(rel)) {
      return { ok: true, scope: "own" };
    }
  }
  // Explicit deny patterns
  for (const pat of entry.deny ?? []) {
    if (globToRegex(pat).test(rel)) {
      return {
        ok: false,
        reason: "cross_runtime_write",
        runtime: runtimeId,
        path: rel,
        hint: `Path '${rel}' is owned by another runtime. See .loop/r2-allowlist.json.`,
      };
    }
  }
  // Default deny (not in own, not in universal)
  return {
    ok: false,
    reason: "cross_runtime_write",
    runtime: runtimeId,
    path: rel,
    hint: `Path '${rel}' is not in runtime '${runtimeId}' allowlist. ` +
          `Add to .loop/r2-allowlist.json if intentional.`,
  };
}

### Path-field detection

```js
// core/r2/path-field-detector.js
//
// Detect which string fields in a tool's inputSchema are file paths.
// Convention: any field with name matching /file_?path|path|file|directory|dir|filename/i
// AND typed as string is treated as a path field.
//
// Future hardening: extend Zod with z.path() for explicit declaration.

const PATH_FIELD_NAMES = /^(file_?path|file|path|directory|dir|filename|evidence_?code_?ref|evidence_?test)$/i;

export function collectPathFields(inputSchema) {
  const fields = [];
  if (!inputSchema?.shape) return fields;
  for (const [name, def] of Object.entries(inputSchema.shape)) {
    if (typeof def?._def?.typeName === "string" && def._def.typeName === "ZodString") {
      if (PATH_FIELD_NAMES.test(name)) {
        fields.push(name);
      }
    }
  }
  return fields;
}
```

### createLoopTool wrapper extension (REVISED — uses findProjectRoot + catches PathContainmentError, red-team Findings 5 + 21)

```js
// tools/learning-loop-mastra/mastra/create-loop-tool.js (after Phase 1 + Phase 2)
import { resolveInsideRoot, PathContainmentError } from "../core/path-containment.js";

export function createLoopTool({ id, description, inputSchema, execute }) {
  const normalized = attachParityJSONSchema(normalizeInputSchema(inputSchema));
  const pathFields = collectPathFields(normalized);
  return createTool({
    id, description, inputSchema: normalized,
    execute: async (args, context) => {
      // Phase 1: identity verifier
      const decision = await verifyRuntimeToken({ tokenB64: ..., expectedRuntimeId: ... });
      if (decision.decision !== "ok") {
        throw new Error(`caller-identity:${decision.decision}`);
      }
      // Phase 2 + Phase 3: R2 ownership + path containment on every path field
      for (const f of pathFields) {
        const v = args?.[f];
        if (typeof v !== "string") continue;
        let absPath;
        try {
          // RED-TEAM FINDING 5: pass _projectRoot (from findProjectRoot()), NOT process.cwd().
          absPath = resolveInsideRoot(v, _projectRoot);
        } catch (err) {
          // RED-TEAM FINDING 21: explicit catch for PathContainmentError → structured envelope
          if (err instanceof PathContainmentError) {
            throw new Error(`path-containment:${err.code}:${v}`);
          }
          throw err;
        }
        const r2 = checkR2Ownership(decision.runtime_id, absPath, _allowlist, _projectRoot);
        if (!r2.ok) {
          throwR2Error(r2, id);
        }
      }
      return execute(args, context);
    },
  });
}

- **R3 (MED, REVISED — red-team Finding 13):** `tools/learning-loop-mastra/**` was REMOVED from `universal` (now in `protected_paths` per §"Allowlist file shape"). - **R4 (MED, REVISED — red-team Finding 14):** Hot-reload not supported — operator deny edits take effect on next MCP server restart. Mitigation: (a) document the window in operator docs; (b) add `meta_state_signal_restart` MCP tool in follow-up plan; (c) log every allowlist edit to gate_log so operator knows deny is pending.
- **R5 (REMOVED — red-team Finding 12):** `minimatch` is NOT in dep tree. **Mitigation applied:** use built-in `RegExp` translation via `globToRegex()` helper; removed `import { minimatch }` from plan.
- **R6 (LOW):** Test runner path injection (Phase 3 covers). Out of scope to fix `verification-runner.js` itself.
